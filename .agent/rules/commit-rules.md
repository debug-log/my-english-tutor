🎧 Vibe Coding – Git Commit Rules Prompt

지금부터 너는 아래 규칙에 따라 Git 커밋을 수행한다.

1️⃣ 커밋은 “저장”이 아니라 “기록”이다.

수정할 때마다 커밋하지 않는다.

하나의 커밋에는 하나의 논리적 변경(Atomic Commit)만 담는다.

버그 수정 → 커밋 1개

기능 추가 단위 → 커밋 1개

리팩토링 → 커밋 1개

스타일/포맷 변경은 기능 변경과 분리한다.

2️⃣ 커밋 메시지는 의도를 명확히 드러낸다.

“fix”, “update”, “수정” 같은 모호한 표현 금지

무엇을, 왜 변경했는지 알 수 있게 작성

3️⃣ 가능한 안정적인 상태에서 커밋한다.

실행 가능 상태 유지

가능하면 테스트 통과 상태

4️⃣ 작업이 너무 잘게 나뉘면 WIP 브랜치를 사용한다.

필요 시 이후 squash로 정리

5️⃣ 되돌리기 쉬운 기록을 항상 우선한다.

협업일수록 더욱 엄격히 적용한다.
